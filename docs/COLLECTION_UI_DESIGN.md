# Hermes - Collection UI Design Specification

> Comprehensive Web UI design for the data collection setup wizard.
>
> Key principle: **NON-DEVELOPERS configure collection via dropdowns, toggles,
> and test buttons in a step-by-step wizard.** No YAML editing, no JSON typing,
> no command-line interaction.

---

## Design Philosophy

```
1. PROGRESSIVE DISCLOSURE
   - Show only what's relevant to the selected source type
   - Hide advanced options behind expandable sections
   - Defaults pre-filled for the most common case

2. INSTANT FEEDBACK
   - [Test Connection] at every step that touches external systems
   - Live preview of matched files / records / responses
   - Validation errors shown inline, never after submission

3. GUIDED, NOT GATED
   - Wizard steps are navigable (click any completed step to return)
   - Draft auto-saved every 30 seconds
   - Can skip optional sections and come back later

4. LANGUAGE
   - All labels in Korean (운영자 대상)
   - Technical identifiers (JSON paths, glob patterns) shown as-is
   - Tooltips explain every non-obvious field
```

## 5. Interaction Patterns

### 5.1 Conditional Field Visibility

Fields appear/disappear based on parent selections. No page reload required.

```
Source Type = "REST API"
  -> Show: URL, Method, Headers, Auth
  Auth Type = "OAuth 2.0"
    -> Show: Token URL, Client ID, Client Secret, Scope
    -> Hide: Token field, API Key fields, Basic Auth fields
  Auth Type = "Bearer Token"
    -> Show: Token field
    -> Hide: OAuth fields, API Key fields
  Method = "POST"
    -> Show: Request Body editor
  Method = "GET"
    -> Hide: Request Body editor
```

### 5.2 Validation Strategy

```
Level 1: Field-level (on blur)
  - Required fields
  - Format validation (URL, hostname, port range)
  - Range validation (min/max for numbers)

Level 2: Section-level (on section complete)
  - Cross-field validation (e.g., archive path != source path)
  - Connection test availability

Level 3: Step-level (on "Next")
  - All required fields filled
  - Connection test passed (for remote sources)

Level 4: Activation-level (on "Activate")
  - All steps valid
  - At least one target configured
  - Pipeline has at least Collect stage
  - Test connection passed within last 30 minutes
```

### 5.3 Error States

### 5.4 Auto-Save Behavior

```
Auto-save triggers:
  - Every 30 seconds if form is dirty
  - On step navigation (before leaving current step)
  - On browser tab blur (losing focus)

Auto-save indicator (top right of wizard):
  "Draft saved 10s ago"  |  "Saving..."  |  "Unsaved changes"

Recovery:
  - On wizard re-open, load last auto-saved draft
  - Show confirmation: "이전에 저장하지 않은 작업이 있습니다. 이어서 진행하시겠습니까?"
  - [Continue Draft]  [Start New]
```
